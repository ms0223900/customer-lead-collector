### US-06：系統會記錄表單送出事件

**作為** 管理者  
**我想要** 系統在表單送出成功或失敗時記錄事件  
**以便** Dashboard 可以計算轉換率，也能觀察表單錯誤

**輸入格式**：
- 表單送出結果（成功寫入 `leads` 或驗證／寫入失敗）
- `session_id`、頁面路徑
- 成功時：`lead_id`（新建立名單的 uuid）
- 失敗時：錯誤原因（可放入 `metadata`）

**輸出格式**：
- 成功：`analytics_events` 一筆，`event_type` = `form_submit_success`，含 `lead_id`
- 失敗：`analytics_events` 一筆，`event_type` = `form_submit_error`，`metadata` 含錯誤欄位或原因

**驗收條件**：
- [ ] 表單送出成功時，建立 `form_submit_success` 事件
- [ ] 表單送出失敗（驗證錯誤或 DB 錯誤）時，建立 `form_submit_error` 事件
- [ ] 成功事件可關聯 `lead_id`
- [ ] 事件包含 `page`、`session_id`、`created_at`
- [ ] 錯誤事件可在 `metadata` 記錄錯誤原因或失敗欄位
- [ ] Dashboard 的名單數與轉換率可使用這些事件或 `leads` 資料計算

**依賴關係**：
- US-04、US-05：表單送出與名單寫入流程
- Sprint 0：`analytics_events` 資料表

**優先級**：P0  
**相關功能**：簡易事件追蹤（spec §6.4）、Dashboard 轉換率（spec §6.5）
