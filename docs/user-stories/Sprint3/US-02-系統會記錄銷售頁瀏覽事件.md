### US-02：系統會記錄銷售頁瀏覽事件

**作為** 管理者  
**我想要** 系統在訪客進入銷售頁時記錄瀏覽事件  
**以便** 後續可以在 Dashboard 中看到總瀏覽數

**輸入格式**：
- 訪客進入 `/` 的 page load 事件
- Client 端產生或讀取的 `session_id`（cookie / localStorage）
- 已建立的 `analytics_events` 資料表（spec §7）

**輸出格式**：
- 一筆寫入 Supabase `analytics_events` 的紀錄：
  - `event_type`: `page_view`
  - `page`: `/`（或 spec 定義之路徑）
  - `session_id`: 訪客識別字串
  - `created_at`: 事件時間

**驗收條件**：
- [ ] 訪客進入銷售頁時，系統建立一筆 `analytics_events` 資料
- [ ] 事件的 `event_type` 為 `page_view`
- [ ] 事件包含 `page` 欄位（例如 `/`）
- [ ] 事件包含 `session_id` 或其他簡易訪客識別
- [ ] 事件包含 `created_at`
- [ ] 重新整理頁面會新增一次瀏覽事件（每次 page load 一筆）

**依賴關係**：
- Sprint 0：`analytics_events` 資料表已建立
- US-01：銷售頁路由存在

**優先級**：P0  
**相關功能**：簡易事件追蹤（spec §6.4）、Dashboard 總瀏覽數（spec §6.5）
