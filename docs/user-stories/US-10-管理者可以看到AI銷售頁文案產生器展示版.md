### US-10：管理者可以看到 AI 銷售頁文案產生器展示版

**作為** 管理者  
**我想要** 在後台使用 AI 銷售頁文案產生器的展示版  
**以便** 理解未來可以如何用 AI 快速產生課程銷售頁文案

**輸入格式**：
- 管理者填寫的輸入欄位（spec §6.6）：
  - 課程名稱
  - 目標學員
  - 核心痛點
  - 課程承諾
  - 語氣風格
- **不**需要 AI API Key；後端回傳固定 Mock 結果

**輸出格式**：
- 介面顯示 Mock 文案區塊：
  - Hero 標題、副標
  - 痛點段落
  - 課程承諾段落
  - CTA 文案
  - FAQ 草稿
- 可選：寫入 `landing_page_copy`，`source` = `mock`
- 頁面醒目標示：「此版本為展示版，文案為 Mock 產生，尚未串接真實 AI API」

**驗收條件**：
- [ ] 後台有「AI 銷售頁文案產生器」區塊或頁籤
- [ ] 頁面清楚標示展示版、尚未串接真正 AI API
- [ ] 管理者可填寫所有輸入欄位
- [ ] 點擊「產生文案」後顯示固定範例或 Mock 結果（含 loading 狀態）
- [ ] 結果內容符合本課程銷售主題
- [ ] 結果可複製或呈現為可套用到 Landing Page 的格式
- [ ] 程式碼與 UI 不會讓使用者誤以為已串接真正 AI
- [ ] 不需任何 AI API Key 即可運作

**依賴關係**：
- Sprint 0：後台 `/admin` 路由；可選 `landing_page_copy` 資料表
- US-07：後台框架（建議同區域新增頁籤）

**優先級**：P1（Should Have）  
**相關功能**：AI 銷售頁文案產生器 Mock 版（spec §6.6、§4 Should Have）

**重要備註**：
- 本 US 為**教學展示版**，不串接真正 AI API。真正 API 串接見 US-12。
