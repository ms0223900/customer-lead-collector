# 客戶名單收集系統｜User Stories 總覽

本目錄收錄《聊著聊著，AI 就做完了》課程 MVP 的正式 User Stories，對應 [`docs/spec.md`](../spec.md) 與 Notion [US 設計](https://app.notion.com/p/64a1c41c17494fae8ed5069afda11535) 頁面。

**設計原則**：
1. 先完成「課程銷售頁 → 等候名單 → 後台查看 → Dashboard」最小閉環
2. AI 文案產生器先做展示版（US-10），真正 API 另拆 US-12
3. Dashboard 指標必須來自資料庫，不得前端假資料
4. 每張 US 可獨立挑選、實作、驗收與 Commit

製作與 Sprint 對照流程見 [`US-製作指引.md`](./US-製作指引.md)。

## 建議開發順序（Build Sprint 對照）

### Wave 0 — 初始化（非正式 US）
Sprint 0：專案、`spec.md`、資料表（`leads`、`analytics_events`）

### Wave 1 — 看得到、填得了（Sprint 1）
- **平行可選**：無
- **順序**：US-01 → US-03 → US-04  
- **目標**：Landing Page + 表單（可先不寫 DB）

### Wave 2 — 存得起、看得到名單（Sprint 2）
- **順序**：US-05 → US-07  
- **目標**：名單持久化 + 後台列表

### Wave 3 — 數據有來源（Sprint 3）
- **順序**：US-02 → US-06 → US-09（US-02/06 可與 US-05 部分重疊實作）  
- **目標**：事件追蹤 + Dashboard 真實統計

### Wave 4 — 名單管理深化（Sprint 4）
- **順序**：US-08  
- **目標**：狀態與備註編輯

### Wave 5 — AI 展示版（Sprint 5）
- **順序**：US-10 →（可選）US-11  
- **目標**：Mock 文案產生器，不串 API

### Wave 6 — 後續迭代（Sprint 6）
- **順序**：US-12  
- **目標**：真正 AI API（非第一版影片主線）

---

## 課程展示建議主線

若只做一支 Demo 影片，建議完成：**Sprint 1 → 2 → 3 → 5**（可口頭帶過 Sprint 0、4、6）。

詳細 Sprint 目標、驗收檢查、AI 指令與 Commit 範例見 [`US-製作指引.md`](./US-製作指引.md)。
