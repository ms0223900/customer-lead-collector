### US-D09：設計 AI 文案產生器展示版

**作為** 負責 MVP 圖面設計的設計師  
**我想要** 完成 AI 銷售頁文案產生器（Mock 展示版）的設計稿  
**以便** 開發 Sprint 5（US-10）時輸入表單、Mock 結果與警語有明確對照，且不讓使用者誤以為已串真實 API

**輸入格式**：
- US-D01 元件（含警告色橫幅樣式）
- US-D05 後台框架
- [`docs/spec.md`](../../spec.md) §6.6 AI 文案產生器輸入與 Mock 輸出欄位

**輸出格式**：
- 嵌入 US-D05「AI 文案產生器」頁籤的設計稿
- **醒目橫幅**：「此版本為展示版，文案為 Mock 產生，尚未串接真實 AI API」
- **輸入區**：課程名稱、目標學員、核心痛點、課程承諾、語氣風格、「產生文案」按鈕
- **結果區**（產生後顯示）：
  - Hero 標題、副標
  - 痛點段落
  - 課程承諾段落
  - CTA 文案
  - FAQ 草稿
- 狀態：空結果 → Loading → 顯示 Mock 文案
- 可選：「複製」按鈕

**驗收條件**：
- [ ] 展示版警語視覺上無法忽略（與一般內容區分明）
- [ ] 輸入區含 spec §6.6 全部欄位
- [ ] 結果區含 Hero、痛點、承諾、CTA、FAQ 區塊
- [ ] Loading 狀態有設計
- [ ] 整體視覺不暗示已為正式 AI 產品（避免誤導）
- [ ] 稿面標註對應實作 US：US-10

**依賴關係**：
- US-D01：元件與 Design Tokens
- US-D05：後台框架

**優先級**：P1  
**相關功能**：AI 銷售頁文案產生器 Mock 版（spec §6.6）

**備註**：
- **Sprint 5 前**完成即可；若僅實作 Sprint 1 + Sprint 2，**可略過**。
- 真正 AI API（US-12）複用本稿 UI，僅替換警語與 loading 語意。
