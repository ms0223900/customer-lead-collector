### US-D02：設計課程銷售頁（桌機版）

**作為** 負責 MVP 圖面設計的設計師  
**我想要** 完成桌機寬度的課程銷售 Landing Page 完整長頁設計稿  
**以便** 開發 Sprint 1（US-01、US-03）時有明確版面與文案區塊可對照

**輸入格式**：
- US-D01 元件與 Design Tokens
- [`docs/spec.md`](../../spec.md) §6.1 課程銷售 Landing Page 區塊定義
- 課程名稱與預設文案方向（spec §6.1）

**輸出格式**：
- 一張可捲動的**桌機版** Landing Page 設計稿（建議寬度 `≥1024px`）
- 區塊由上到下至少包含：
  1. **Hero**：課程名稱、一句話價值主張、「加入等候名單」CTA
  2. **痛點區**：至少 3 個目標學員痛點
  3. **課程承諾**：明確學習成果
  4. **適合對象**
  5. **課程內容摘要**（選題、Spec、Rules、Build、Debug、Deploy 等）
  6. **等候名單表單區**（嵌入頁尾，Default 狀態即可；細部狀態見 US-D04）
- Hero CTA 行為註解：點擊後 smooth scroll 至表單區（對應 US-03）

**驗收條件**：
- [x] 顯示完整課程名稱（spec §6.1）
- [x] 顯示一句話價值主張
- [x] 痛點區至少 3 項
- [x] 含課程承諾、適合對象、課程內容摘要區塊
- [x] Hero 與頁尾（或表單區上方）有可辨識的「加入等候名單」CTA
- [x] 表單區塊位置明確，可供 CTA 捲動定位
- [x] 視覺風格引用 US-D01 元件
- [x] 稿面標註對應實作 US：US-01、US-03

**依賴關係**：
- US-D01：元件與 Design Tokens

**優先級**：P0  
**相關功能**：課程銷售 Landing Page（spec §6.1、§8 訪客前台）
