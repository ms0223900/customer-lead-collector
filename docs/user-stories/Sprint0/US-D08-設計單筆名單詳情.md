### US-D08：設計單筆名單詳情

**作為** 負責 MVP 圖面設計的設計師  
**我想要** 完成單筆名單詳細資料檢視與編輯的設計稿  
**以便** 開發 Sprint 4（US-08）時 Modal／Drawer、狀態與備註編輯有明確對照

**輸入格式**：
- US-D01 表單元件、Status Badge
- US-D06 名單列表（點擊列進入詳情）
- [`docs/spec.md`](../../spec.md) §6.3 單筆詳情與狀態選項

**輸出格式**：
- **Modal 或 Drawer** 設計稿，含：
  - **唯讀區**：姓名、Email、身分、產品想法、卡關點、訪談意願、建立時間
  - **編輯區**：狀態下拉（新名單／高意願／已聯絡／已訪談／不適合）、備註 textarea（可空）、儲存按鈕
- 狀態變體（可註解或分格）：
  - 儲存中 loading
  - 儲存失敗錯誤提示
  - 儲存成功（toast 或 inline 回饋）

**驗收條件**：
- [ ] 唯讀區含 US-08 要求之全部欄位
- [ ] 狀態下拉含五種選項
- [ ] 備註可為空仍有明確輸入區
- [ ] 儲存按鈕與 loading／錯誤狀態有設計
- [ ] 與 US-D06 列表列點擊行為銜接（註解說明）
- [ ] 稿面標註對應實作 US：US-08

**依賴關係**：
- US-D01：元件與 Design Tokens
- US-D06：後台名單列表

**優先級**：P1  
**相關功能**：名單管理（spec §6.3）

**備註**：
- **Sprint 4 前**完成即可；若僅實作 Sprint 1 + Sprint 2，**可略過**。
