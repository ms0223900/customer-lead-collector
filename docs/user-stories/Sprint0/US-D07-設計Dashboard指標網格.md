### US-D07：設計 Dashboard 指標網格

**作為** 負責 MVP 圖面設計的設計師  
**我想要** 完成後台 Dashboard 七項指標卡片的版面設計稿  
**以便** 開發 Sprint 3（US-09）時指標呈現與排版有明確對照

**輸入格式**：
- US-D01 Metric Card 元件（若 US-D01 尚未定義，本稿需一併產出卡片樣式）
- US-D05 後台框架
- [`docs/spec.md`](../../spec.md) §6.5 七項指標定義

**輸出格式**：
- 嵌入 US-D05 主內容區的 **Dashboard** 設計稿
- **桌機**：3–4 欄 card grid
- **手機**（建議）：單欄堆疊
- 七張指標卡，標題與數字區域明確：
  1. 總瀏覽數
  2. CTA 點擊數
  3. 總名單數
  4. 轉換率（`%`；分母為 0 時顯示 `0%`）
  5. 今日新增名單
  6. 最近 7 天新增名單
  7. 高意願名單數（可加小字說明計算規則）

**驗收條件**：
- [ ] 七項指標皆有獨立卡片與繁中標題
- [ ] 轉換率卡片可容納百分比顯示（含 `0%` 情境）
- [ ] 桌機與手機排版皆有稿（或 responsive 註解）
- [ ] 數字區域留彈性，不寫死假數據（實作自 DB 計算）
- [ ] 嵌入 US-D05 導覽中「Dashboard」位置
- [ ] 稿面標註對應實作 US：US-09

**依賴關係**：
- US-D01：元件與 Design Tokens（Metric Card）
- US-D05：後台框架

**優先級**：P1  
**相關功能**：管理者 Dashboard（spec §6.5、§8）

**備註**：
- **Sprint 3 前**完成即可；若僅實作 Sprint 1 + Sprint 2，**可略過**。
- spec Could Have「最近 7 天名單趨勢圖」不在本 US；若要做可另開設計備註。
