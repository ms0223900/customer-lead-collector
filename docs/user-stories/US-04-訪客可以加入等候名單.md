### US-04：訪客可以加入等候名單

**作為** 一位對課程有興趣的訪客  
**我想要** 填寫並送出等候名單表單  
**以便** 課程開放時可以收到通知

**輸入格式**：
- 表單欄位（spec §6.2）：
  - `name`：姓名（選填）
  - `email`：Email（**必填**）
  - `role`：身分 / 角色（選填）
  - `product_idea`：最想用 AI 做出的產品（選填）
  - `pain_point`：目前卡關點（選填）
  - `interview_intent`：是否願意接受訪談（布林，預設 `false`）

**輸出格式**：
- 驗證通過：表單區塊替換為成功訊息（不跳頁），文案為 spec §6.2 定義之繁中感謝訊息
- 驗證失敗：欄位下方顯示繁中錯誤提示（Email 空白或格式錯誤）
- UI 狀態：`loading` → `success` 或 `error`

**驗收條件**：
- [ ] 表單顯示所有欄位（姓名、Email、身分、產品想法、卡關點、訪談意願）
- [ ] Email 為必填；未填或格式錯誤（缺少 `@` 等）無法送出
- [ ] Email 格式錯誤時，欄位下方顯示繁中提示
- [ ] 送出中顯示 loading 狀態，避免重複提交
- [ ] 送出成功後顯示 spec 定義的成功訊息，使用者可理解已加入等候名單
- [ ] 送出失敗（如 API 錯誤）時顯示可讀繁中錯誤訊息，頁面不崩潰
- [ ] 手機尺寸可正常操作表單

**依賴關係**：
- US-01：Landing Page 與表單區塊
- US-03：CTA 導向表單（建議同 Sprint 完成）

**優先級**：P0  
**相關功能**：等候名單表單（spec §6.2、§8 訪客前台）

**備註**：
- Sprint 1 可先只做前端表單與驗證，資料持久化由 US-05 負責。
