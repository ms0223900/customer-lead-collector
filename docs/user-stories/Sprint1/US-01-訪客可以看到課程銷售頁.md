### US-01：訪客可以看到課程銷售頁

**作為** 一位對 AI Side Project 課程有興趣的訪客  
**我想要** 看到完整的課程銷售 Landing Page  
**以便** 快速理解這堂課是否能解決我的問題

**輸入格式**：
- 路由 `/` 的頁面請求
- 靜態或預設的課程文案內容（對應 [`docs/spec.md`](../spec.md) §6.1）
- 課程名稱：《聊著聊著，AI 就做完了：不懂程式也能用 AI 做出可上線產品》

**輸出格式**：
- 可閱讀的 Landing Page，包含 Hero、痛點、課程承諾、適合對象、課程內容摘要、等候名單 CTA 等區塊
- 桌機與手機寬度下版面正常顯示

**驗收條件**：
- [x] 頁面顯示完整課程名稱
- [x] 頁面顯示一句話價值主張
- [x] 頁面顯示至少 3 個目標學員痛點
- [x] 頁面顯示課程可帶走的學習成果（課程承諾）
- [x] 頁面顯示適合對象說明
- [x] 頁面顯示課程內容摘要（選題、Spec、Rules、Build、Debug、Deploy 等）
- [x] 頁面顯示「加入等候名單」CTA
- [x] 桌機與手機寬度皆可正常閱讀，無明顯版面錯位

**依賴關係**：
- Sprint 0 專案初始化（Next.js 專案可啟動）

**優先級**：P0  
**相關功能**：課程銷售 Landing Page（spec §6.1、§8 訪客前台）
