### US-12：管理者可以串接真正 AI API 產生銷售頁文案

**作為** 管理者  
**我想要** 串接真正 AI API，根據輸入資訊產生銷售頁文案  
**以便** 未來可以用真實 AI 輔助建立銷售頁

**輸入格式**：
- 與 US-10 相同之輸入欄位（課程名稱、目標學員、核心痛點、課程承諾、語氣風格）
- 後端環境變數中的 AI API Key（**不得**暴露至 client）

**輸出格式**：
- 後端 Route Handler 呼叫 AI API，回傳結構化文案（Hero、副標、痛點、承諾、CTA、FAQ）
- 可選：寫入 `landing_page_copy`，`source` = `ai_api`
- 失敗時：繁中錯誤訊息；成功時：loading → 結果展示

**驗收條件**：
- [ ] AI API Key 僅透過後端環境變數讀取，不出現在 `NEXT_PUBLIC_*` 或前端 bundle
- [ ] 管理者送出輸入後，後端呼叫 AI API 並回傳文案
- [ ] 空白必填輸入有基本防呆，不可直接送出
- [ ] 請求進行中顯示 loading 狀態
- [ ] API 失敗時顯示可讀繁中錯誤訊息
- [ ] 成功結果格式與 US-10 輸出結構相容

**依賴關係**：
- US-10：展示版 UI 與輸入／輸出格式可複用
- 外部：AI 服務 API Key 與配額

**優先級**：P2（後續迭代；**不屬於** MVP 教學展示主線）  
**相關功能**：spec §4 Won't Have「真正 AI API 串接」之後續擴充

**重要備註**：
- 本 US **不**列入第一支展示影片主線；應在 MVP 驗收完成後另開 Sprint 實作。
