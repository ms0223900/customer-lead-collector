### US-08：管理者可以查看與編輯單筆名單狀態

**作為** 管理者  
**我想要** 查看單筆名單詳細資料並更新狀態與備註  
**以便** 區分新名單、高意願名單與已聯絡名單

**輸入格式**：
- 列表中選取的名單 `id`
- 更新 payload：`status`（新名單 / 高意願 / 已聯絡 / 已訪談 / 不適合）、`note`（選填）

**輸出格式**：
- 詳細檢視：姓名、Email、身分、產品想法、卡關點、訪談意願、狀態、備註、建立時間
- 儲存成功後：`leads` 紀錄更新，`updated_at` 刷新；列表反映新狀態

**驗收條件**：
- [ ] 管理者可從列表點擊查看單筆名單詳細內容（詳情頁或彈窗）
- [ ] 詳細內容包含姓名、Email、身分、產品想法、卡關點、訪談意願
- [ ] 管理者可更新狀態；選項至少包含：新名單、高意願、已聯絡、已訪談、不適合
- [ ] 管理者可新增或修改備註（備註可空）
- [ ] 按下儲存後，重新整理後台仍保留更新內容
- [ ] 儲存失敗時顯示繁中錯誤訊息

**依賴關係**：
- US-07：名單列表與後台入口

**優先級**：P0（spec Must Have）；Build Sprint 4 可延後實作  
**相關功能**：名單管理（spec §6.3）、Dashboard 高意願名單數（spec §6.5）
